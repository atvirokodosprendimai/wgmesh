## Research & Outreach Notes

### Outreach Best Practices

1. **Join first, promote later**
   - Join communities 1-2 weeks before any promotional posts
   - Participate in discussions, help others with questions
   - Establish presence as helpful community member

2. **Respect self-promotion rules**
   - Review community guidelines before posting
   - Contact moderators if unclear about promotion policies
   - Some communities require prior approval for promotional posts

3. **Provide genuine value**
   - Share knowledge about mesh networking, WireGuard, NAT traversal
   - Answer technical questions about networking and infrastructure
   - Contextualize wgmesh/cloudroof as solution to real problems

4. **Customize messaging per community**
   - Kubernetes communities: emphasize cluster mesh, pod-to-pod networking
   - Homelab communities: emphasize VPS-to-home connectivity
   - Security communities: emphasize encryption, no-cloud-trust architecture
   - Developer communities: emphasize API, CLI, automation

### Community Engagement Flow

```
1. Join community
   ↓
2. Introduce yourself personally (not as brand)
   ↓
3. Participate in discussions for 1-2 weeks
   ↓
4. Identify appropriate channel/context for trial offer
   ↓
5. Post trial offer with clear value proposition
   ↓
6. Follow up on questions and support requests promptly
```
