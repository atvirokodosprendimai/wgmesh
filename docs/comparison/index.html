wgmesh vs Tailscale

wgmesh is a decentralized WireGuard mesh builder for edge/IoT deployments.
Tailscale is a centralized NAT traversal service for remote access.

Quick Comparison

Aspect wgmesh Tailscale
Architecture Decentralized mesh, DHT-based discovery Centralized coordination server, DERP relays
Trust Model Shared secret, no central broker Account-based, OAuth/OIDC to coordination server
Self-Hosted Yes (Go binary, single executable) Partial (head-scale available, but limited)
Privacy No data collection, Dandelion++ relay Coordination server logs connections, relay usage
Use Case Edge/IoT fleets, air-gapped networks Remote access, small teams, mobile-first
Cost Infrastructure only (self-hosted) Subscription tiers, per-user/per-device pricing

Architecture

wgmesh: Decentralized Mesh

  • Discovery: Multi-layer DHT, LAN multicast, GitHub Issues bootstrap
  • Routing: Full mesh topology, peer-to-peer connections
  • Relays: Dandelion++ for announcement privacy
  • State: Local peer store, automatic reconciliation
  • No single point of failure — mesh continues operating without central servers

Tailscale: Centralized Coordination

  • Coordination Server: All nodes register to control.tailscale.com
  • Discovery: NAT traversal via DERP relays, UDP hole-punching
  • Authentication: OAuth/OIDC (Google, GitHub, Microsoft)
  • Relays: DERP servers operated by Tailscale
  • Coordination server required — unavailable server = no mesh updates

Security & Privacy

Aspect wgmesh Tailscale
Trust Anchor Shared secret (HKDF-derived keys) OAuth/OIDC identity provider
Key Distribution Automatic via DHT, encrypted envelopes Centralized server, client fetches peer list
Compromise Impact Secret compromise affects mesh only Coordination server compromise affects all meshes
Auditability Open-source codebase, fully auditable Closed-source control plane (DERP, coordination logic)
Data Collection None — discovery over public DHT (anonymized) Coordination server logs connections, relay usage
Relay Privacy Dandelion++ (randomized paths, source hiding) DERP paths visible to coordination server

Deployment & Operations

Choose wgmesh If:

  • Deploying large edge/IoT fleets with intermittent connectivity
  • Operating in air-gapped networks without internet access
  • Compliance requirements demand full control over data
  • You prefer self-hosting and owning the entire stack
  • Cost sensitivity — large fleets (>100 nodes)
  • Need open-source code for security review

Example: 500-node IoT sensor network across regions, air-gapped SCADA system

Choose Tailscale If:

  • Need quick remote access for non-technical users
  • Small teams (<20 users) with simple access control
  • Mobile-first deployment (iOS, Android native apps)
  • Existing OAuth/OIDC provider integration
  • Prefer managed service over self-hosted

Example: 5-person remote team, individual developer accessing home lab
